# Project-Specific Guidance

This file holds repo-local conventions that don't belong in the keelcore
standards submodule. It is loaded automatically via `@.local-claude.md`
from `CLAUDE.md` (the canonical adapter at
`.standards/adapters/claude/CLAUDE.md`).

Created by bootstrap-standards.sh template. Replace this content with
project-specific guidance: workspace shape, build instructions, test
layout, commit policy, runtime context.

## Suggested sections

- **Workspace shape** — language, crate/module layout, where binaries live.
- **Build & data** — primary `make` targets, runtime data sources, log
  conventions, env vars.
- **Tests** — which test runner covers what; how integration tests are
  invoked; fixture layout.
- **Commit policy** — DCO requirement (`git commit -s`), branch naming,
  PR template, who owns commits.
- **Canonical scripts** — which `scripts/` are upstream-byte-identical
  vs. project-specific.
- **Pre-commit** — `make install-hooks` to wire the hook locally.

## Standards inheritance

All other engineering standards inherit from `.standards/governance/*.md`.
Do not duplicate that content here — link to it from `CLAUDE.md` if needed.
